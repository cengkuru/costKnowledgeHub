## Key Information Architecture

### The CoST Framework (3 Pillars)

1. **Disclosure**
   - Publication of infrastructure project data
   - 40 CoST IDS elements across project lifecycle
   - Formats: CoST IDS or OC4IDS
   - Resources: 7+ guidance documents

2. **Assurance**
   - Independent verification of published data
   - Data quality review
   - Issue identification and analysis
   - Transformation to compelling information
   - Resources: 3+ guidance documents + impact studies

3. **Multi-Stakeholder Engagement**
   - Government participation
   - Private sector involvement
   - Civil society engagement
   - Social accountability mechanisms
   - Resources: Case studies from 5+ countries

### The Data Standards Stack

```
Project Level: CoST Infrastructure Data Standard (IDS)
├─ 40 data elements
├─ Project lifecycle coverage
└─ Project identification → preparation → implementation → completion

Contract Level: Open Contracting Data Standard (OCDS)
├─ Procurement process details
└─ Contracting information

Integration: OC4IDS (Open Contracting for Infrastructure Data Standard)
├─ Combines project + contract data
├─ Integrated approach
└─ Single standardized format
```

### Data Elements Coverage

**Project Identification:** Project name, description, location, sector, value

**Project Preparation:** Feasibility studies, design cost estimates, timeline

**Project Procurement:** Tender details, contract award, supplier information

**Project Implementation:** Budgets, progress, milestones, changes

**Project Completion:** Final cost, timeline variance, performance metrics

## Implementation Models Featured

### 1. Direct Entry Model
**Examples:** CoST Honduras, West Lombok (Indonesia)
- Direct data entry to transparency portal
- Portal publishes in OC4IDS format
- Organizational structure: Single portal with procuring entity data input

### 2. Extended Procurement System Model
**Examples:** Uganda PPDA, Government Procurement Portal
- Extends existing procurement system
- Collects both project and contract data
- System publishes in OC4IDS and OCDS format

### 3. Hybrid Model
**Examples:** Various CoST programs
- Project data in transparency portal
- Contract data imported from procurement system
- Combined publication in OC4IDS format
